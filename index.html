Network visualizations

This page shows several visualizations of the same network, generated using different visualization libraries. You can find the scripts for generating these in the GitHub repository.

Sigma.js

Sigma.js is a JavaScript library for visualizing graphs. It has good performance, and is easy to get started with (see their examples), and has very good performance, but is "opinionated", which limits the customization options or integrating it with other features.

HoloViews

HoloViews is a Python library for creating interactive data visualizations, including network graphs. It works nicely with Pandas DataFrames, and is part of the broader Holoviz ecosystem, which enalbes easy integration into other libraries for high-performance data analysis and visualization. It can be difficult to customize beyond the options provided in the library though.

D3

D3 is a JavaScript library for creating interactive data visualizations. It is more of a set of building blocks rather than a complete framework, so you'll need to put together those blocks yourself (or build on examples), The upside is that it's extremely flexible, there are no real limitations on what you can do. Its default performance (with SVG rendering) is not great, but using canvas rendering improves it significantly.

D3 - force-directed

This visualization uses D3 and the same data as the D3 example above. Except instead of using the computed layout (x and y positions) from Gephi, it uses a force-directed layout algorithm to compute the positions of the nodes. This kind of "live" network layout is better when the data changes on a regular basis (and you don't want to manually lay it out in Gephi every update), but typically results in less control of the layout and worse performance

Sigma.js (fancy)

This visualization also uses sigma.js, but it's built on the base of a very nice demo that shows many of the framework's features. The result is very polished, and includes a description, ability to search by node name, and a way to interact with and show clusters/categories, all with quite good performance. The downside is that it's not straightforward to customize.

Choices

During each step of the network visualization process, you have many choices available to you. This visualization lets you explore some of these available choices, for visualizing the same data.

  • Filtering: for visualization purposes, it often makes sense to filter out (remove) particular nodes based on some criteria. The "Appearance Threshold" slider removes guests who have appeared on less than a specificed number of different podcasts. This allows only the most significant guests to be shown.
  • Graph type: a bipartite graph has two different kind of nodes, where there's an implicit relationship between them. A unipartite graph has only one kind of node. The "Graph Type" section lets you switch between a bipartite graph (where there are two kinds of nodes: podcasts and guests, and every edge indicates a guest appearance on a podcast) and a unipartite graph (there are only guests, and every edge indicates two guests have appeared on the same podcast)
  • Display: This section enables you to toggle whether or not to show labels and edges.
